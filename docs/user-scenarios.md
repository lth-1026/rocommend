## 1. 페르소나 정의

### P1. 김민지 — 탐험가 (32세, 직장인)

| 항목 | 내용 |
|------|------|
| 커피 습관 | 매일 핸드드립·에스프레소로 홈카페. 에티오피아 내추럴 과일향 선호. |
| 구매 패턴 | 월 1~2회 온라인 구매. 예산 월 20,000~40,000원. |
| 기존 탐색 방법 | 인스타그램 해시태그, 네이버 블로그, 지인 추천. |
| 핵심 불만 | "리뷰 좋다고 샀는데 내 취향이 아닌 경우가 많다. 나랑 입맛 비슷한 사람이 추천해주면 좋겠다." |
| 세그먼트 | 탐험가 (새로운 로스터리를 적극 탐색, 월 1회 이상 구매) |
| 기기 환경 | 탐색은 모바일, 구매 결정은 PC. |
| roco 사용 목적 | 취향 유사 사용자 기반 새로운 로스터리 발견 및 구매 전환 |

### P2. 박준호 — 루틴 구매자 (38세, 프리랜서 개발자)

| 항목 | 내용 |
|------|------|
| 커피 습관 | 하루 2잔. 주로 에스프레소. 2~3개 로스터리를 번갈아 구매. |
| 구매 패턴 | 월 2회. 가끔 새로운 로스터리를 시도하지만 실패하면 기존 단골로 복귀. |
| 핵심 불만 | "단골 외 새로운 곳을 시도할 때 어디를 골라야 할지 확신이 없다. 검증된 추천이 필요하다." |
| 세그먼트 | 루틴 구매자 (2~3개 단골 + 점진적 탐색) |
| 기기 환경 | 주로 PC, 모바일 병행. |
| roco 사용 목적 | 기존 취향과 일치하는 검증된 새 로스터리 탐색. "또 사고 싶은 로스터리" 재확인. |

### P3. 이지수 — 입문자 (25세, 대학원생)

| 항목 | 내용 |
|------|------|
| 커피 습관 | 홈카페 시작 2개월. 원두를 처음 구매해보는 단계. 드립백 경험만 있음. |
| 구매 패턴 | 첫 온라인 원두 구매를 앞두고 있음. 취향이 아직 불명확. |
| 핵심 불만 | "처음이라 어디서 사야 할지 모르겠다. 실패하면 아깝고, 맛있는 걸 고르고 싶다." |
| 세그먼트 | 입문자 (Q4=FIRST_TIME. 취향 데이터 없음) |
| 기기 환경 | 모바일 중심. |
| roco 사용 목적 | 실패 없는 첫 구매 결정. 인기 로스터리 기반 안정적 진입. |

## 2. 정상 경로 시나리오

---

### 2-A. 탐험가 — 온보딩부터 CF 추천 경험까지 (P1 김민지)

**목표**: 신규 가입 후 온보딩을 완료하고 즉시 CF 추천을 경험하며 외부 구매 링크까지 도달한다.

**사전 조건**: 서비스 첫 방문. Google 계정 보유.

---

**단계 1. 서비스 진입 및 로그인**

1. 사용자가 roco 홈 페이지에 접근한다.
2. "Google로 시작하기" 버튼을 클릭한다.
3. Google OAuth 인증 화면이 열린다.
4. 계정을 선택하고 허용한다.
5. 인증 성공 후 서버가 User 레코드를 신규 생성한다.
   - `User.preferences`는 `null`로 초기화된다.
   - `onboardingVersion`이 `null`이므로 온보딩 설문 화면으로 자동 리디렉션된다.

> **AC 연결**: F-001 — "소셜 로그인 직후 `User.preferences.onboardingVersion`이 3 미만이거나 null인 사용자는 온보딩 설문 화면(F-002)으로 이동한다."

---

**단계 2. 온보딩 설문 응답 (5문항)**

**Q1. 브루잉 방법 (복수 선택, 최소 1개)**

- 사용자가 "핸드드립"과 "에스프레소 기반"을 선택하고 "다음"을 클릭한다.
- `User.preferences.brewingMethod`에 저장된다 (향후 F-011 취향 프로필용. MVP 추천 로직 미사용).

**Q2. 구매 성향 (단일 선택)**

- 사용자가 "탐험 선호"를 선택한다.
- `User.preferences.purchaseStyle = 'EXPLORER'`로 저장된다 (MVP 추천 로직 미사용).

**Q3. 선호 가격대 (복수 선택, 최소 1개)**

- 사용자가 "20,000~35,000원"과 "35,000원 이상"을 선택한다.
- `User.preferences.preferredPriceRange = ['MID', 'HIGH']`로 저장된다.
- 향후 취향 프로필(F-011) 용도로 저장만 유지된다. MVP 폴백 추천 로직에는 사용되지 않는다.

**Q4. 구매 빈도 (단일 선택)**

- 사용자가 "월 1회"를 선택한다.
- `FIRST_TIME`이 아니므로 Q5 화면이 이어서 표시된다.
- `User.preferences.purchaseFrequency = 'MONTHLY'`로 저장된다.

**Q5. 좋아하는 로스터리 선택 (복수, 최소 3개 이상)**

- 큐레이션된 20개 고정 목록이 표시된다.
- 사용자가 "블루보틀 서울", "텀블러커피", "펠트커피" 3곳을 선택한다.
- "제출" 버튼이 활성화된다.

> **AC 연결**: F-002 — "Q5는 복수 선택이 가능하며 최소 3개 이상 선택해야 하며, 3개 미만 선택 시 다음 단계로 진행되지 않는다. 선택 상한은 없다."

---

**단계 3. 온보딩 제출 및 CF 즉시 활성화**

1. "제출" 버튼 클릭 시 서버가 다음을 처리한다:
   - Q5에서 선택한 3개 로스터리를 각각 `score=5, source="onboarding"`으로 Rating 테이블에 저장한다.
   - 3개 로스터리의 `origins[]`, `priceRange`, `region` 속성을 추출하여 `inferredProfile`을 생성하고 `User.preferences`에 저장한다.
   - `User.preferences.onboardingVersion = 3`, `completedAt = [현재 시각]`을 기록한다.
2. 평가 수가 즉시 3개가 되므로 CF 추천이 즉시 활성화된다. 배치 대기 없음.
3. 홈 피드 페이지로 이동한다.

> **AC 연결**: F-002 — "Q5에서 최소 3개의 로스터리를 선택하고 온보딩을 완료하면, 평가 3개 이상이 즉시 충족되므로 CF가 즉시 활성화되어 홈 피드에 CF 기반 두 섹션이 표시된다."

---

**단계 4. 홈 피드 — CF 추천 확인**

1. 홈 피드에 두 섹션이 표시된다:
   - "새로운 로스터리": CF 추천 결과 중 Rating 이력이 없는 로스터리, 최대 5개.
   - "또 사고 싶은 로스터리": CF 추천 결과 중 온보딩에서 5점 Rating을 저장한 로스터리(블루보틀 서울, 텀블러커피, 펠트커피), 최대 5개.
2. 각 카드에는 로스터리 이름, 지역, 평균 평점이 표시된다.

> **섹션 분류 기준**: Rating 이력 여부로 결정. 온보딩 Q5 선택 = Rating 저장 = "평가한 적 있음" → "또 사고 싶은 로스터리" 섹션에 포함.

---

**단계 5. 추천 클릭 및 상세 페이지 탐색**

1. "새로운 로스터리" 섹션에서 "커피리브레"를 클릭한다.
2. `recommendation_clicked` 이벤트가 로깅된다 (파라미터: userId, roasteryId, recommendationSource="cf", timestamp).
3. 로스터리 상세 페이지가 열린다:
   - 이름, 지역, 가격대, 평균 평점 표시.
   - 판매 중인 원두 목록 (원두명, 원산지, 로스팅 레벨, 디카페인 여부) 표시.
   - 외부 구매 링크 버튼 표시.

---

**단계 6. 외부 구매 링크 클릭 (North Star 달성)**

1. 사용자가 "구매하러 가기" 링크를 클릭한다.
2. `purchase_link_clicked` 이벤트가 로깅된다.
3. 로스터리 외부 사이트가 새 탭으로 열린다.
4. 동일 세션 내 `recommendation_clicked` → `purchase_link_clicked` 연속 발생으로 North Star 지표가 1건 기록된다.

> **North Star 측정**: "recommendation_clicked 이벤트 후 동일 세션 내 purchase_link_clicked 이벤트 발생 여부"

---

### 2-B. 루틴 구매자 — 평가를 쌓아 개인화 추천 고도화 (P2 박준호)

**목표**: 기존 온보딩 완료 사용자가 추가 평가를 남기고, CF 추천이 갱신되는 흐름을 경험한다.

**사전 조건**: 온보딩 완료 상태 (`onboardingVersion=3`). Q5에서 3개 선택 완료 (Rating 3개 보유). CF 활성화 상태.

---

**단계 1. 재방문 및 자동 로그인**

1. 사용자가 브라우저에서 roco를 다시 방문한다.
2. 세션 유효 기간(90일) 이내이므로 로그인 없이 홈 피드가 표시된다.
3. 홈 피드에 "새로운 로스터리" 5개, "또 사고 싶은 로스터리" 3개가 표시된다.

---

**단계 2. 로스터리 탐색 — 필터 활용**

1. 로스터리 목록 페이지로 이동한다.
2. 지역 필터에서 "서울"과 "부산"을 동시 선택한다. (OR 조합: 서울 또는 부산 로스터리)
3. 가격대 필터에서 "MID (20,000~35,000원)"를 선택한다.
4. 결과: 지역=(서울 OR 부산) AND 가격대=MID 인 로스터리 목록 표시.
5. 필터 조건이 URL에 반영된다 (예: `?region=서울,부산&price=MID`).

> **AC 연결**: F-004 — "카테고리 내 OR, 카테고리 간 AND 조건으로 결과 반환"

---

**단계 3. 로스터리 상세 페이지 — 평가 작성**

1. "모모스커피 부산" 카드를 클릭하여 상세 페이지로 이동한다.
2. 사용자가 별점 4점을 선택한다.
3. 한줄평 "부산 여행 중에 갔었는데 에스프레소가 정말 맛있었어요"를 입력한다 (40자, 100자 이내).
4. "평가 저장" 버튼을 클릭한다.
5. 성공 시 토스트 메시지 "평가가 저장되었습니다"가 표시된다.
6. `rating_submitted` 이벤트가 로깅된다 (isFirstRating=true).
7. 해당 로스터리의 평균 평점이 신규 평가를 반영한 값으로 즉시 갱신된다.
8. 서버가 해당 사용자의 CF 추천을 즉시 재계산하여 Recommendation 테이블에 반영한다.

> **AC 연결**: F-005 — "기존 평가를 수정하여 저장하면 변경된 점수가 즉시 평균 평점에 반영된다." / F-006 — "사용자가 평가를 제출하거나 수정하면 다음 홈 피드 로드 시 두 섹션의 구성이 변경된 추천 결과를 반영한다."

---

**단계 4. 홈 피드 재방문 — 갱신된 추천 확인**

1. 홈 피드로 이동하면 CF 추천이 재계산된 상태를 반영한다.
2. "모모스커피 부산"이 4점 Rating이 있으므로 "또 사고 싶은 로스터리" 섹션에 포함된다.
3. "새로운 로스터리" 섹션에서 "모모스커피 부산"은 제외된다.

---

**단계 5. 즐겨찾기 추가**

1. "모모스커피 부산" 상세 페이지에서 즐겨찾기 버튼(하트 아이콘)을 클릭한다.
2. 버튼 상태가 즉시 "추가됨"으로 변경된다.
3. 페이지 새로고침 후에도 "추가됨" 상태가 유지된다.
4. 즐겨찾기 목록 페이지에서 해당 로스터리를 확인한다.

---

### 2-C. 입문자 (처음 구매) — 콜드스타트 폴백 경험 (P3 이지수)

**목표**: Q4에서 "처음 구매해요"를 선택한 사용자가 Q5 없이 4문항으로 온보딩을 완료하고, "실시간 인기 로스터리" 폴백 추천을 받는다.

**사전 조건**: 신규 가입. Kakao 계정 사용.

---

**단계 1. Kakao 로그인 및 온보딩 시작**

1. "Kakao로 시작하기"를 클릭하고 Kakao 인증을 완료한다.
2. 신규 사용자이므로 온보딩 설문으로 자동 이동한다.

---

**단계 2. 온보딩 응답 (4문항)**

### 4-F. CF 빈 배열 — 두 섹션 모두 0개

**시나리오**: CF 추천 계산 결과가 유효하지만, 분류 후 두 섹션 모두 0개인 경우.

**발생 조건**: 해당 사용자가 서비스 내 모든 로스터리를 평가했거나, 유사 사용자 수(K)가 매우 적어 예측 점수를 계산할 수 없는 경우.

1. 홈 피드 요청 시 Recommendation 테이블 조회.
2. "새로운 로스터리" 후보 0개, "또 사고 싶은 로스터리" 후보 0개.
3. 두 CF 섹션(헤더 포함)이 표시되지 않는다.
4. "실시간 인기 로스터리" 섹션이 대신 표시된다 (전체 평균 평점 상위 5개).

> **AC 연결**: F-006 — "두 CF 섹션 모두 결과가 0개인 경우, CF 섹션 대신 '실시간 인기 로스터리' 섹션이 표시된다."

**QA 검증 항목**:

- [ ] "또 사고 싶은 로스터리" 섹션 헤더가 DOM에서 제거된다.
- [ ] "실시간 인기 로스터리" 섹션이 5개 카드와 함께 표시된다.

### 4-G. 평가 삭제로 CF 비활성화 (평가 수 3개 미만으로 하락)

**시나리오**: 온보딩에서 Q5로 3개 Rating 보유 (CF 활성 상태). 이 중 1개를 삭제하여 Rating 수 = 2.

1. 홈 피드 "또 사고 싶은 로스터리" 섹션에 온보딩 선택 로스터리 A, B, C가 있다.
2. 사용자가 로스터리 A 상세 페이지로 이동.
3. 평가 수정 UI에서 "평가 삭제" 클릭 → 확인 다이얼로그.
4. 삭제 확인 → Rating 레코드 즉시 제거.
5. 총 Rating 수 = 2 (3개 미만).
6. 해당 사용자의 CF 즉시 재계산.
7. 다음 홈 피드 로드 시:
   - "새로운 로스터리", "또 사고 싶은 로스터리" CF 섹션이 표시되지 않는다.
   - "실시간 인기 로스터리" 섹션이 표시된다.
8. 로스터리 A 상세 페이지의 평균 평점·평가 수가 즉시 재계산된다.

> **AC 연결**: F-005 — "평가 삭제로 해당 사용자의 총 평가 수가 3개 미만이 되면, 다음 홈 피드 로드 시 CF 섹션이 '실시간 인기 로스터리' 섹션으로 전환된다."

**QA 검증 항목**:
- [ ] Rating 수 = 2로 하락 시 다음 홈 피드 로드에서 CF 섹션이 사라진다.
- [ ] 로스터리 A의 평균 평점이 A의 Rating 제거를 반영하여 갱신된다.
- [ ] 삭제된 Rating이 DB에서 완전히 제거되었는지 확인.

### 4-H. 온보딩 Q5 선택 로스터리 재평가

**시나리오**: 온보딩에서 "블루보틀 서울"을 선택(5점 Rating 저장). 이후 상세 페이지에서 직접 재평가.

1. 사용자가 "블루보틀 서울" 상세 페이지로 이동한다.
2. 평가 버튼 클릭 시 기존 점수 5점이 선택된 상태로 수정 UI가 표시된다.
3. 사용자가 3점으로 변경하고 저장한다.
4. Rating.score가 5에서 3으로 교체된다 (덮어쓰기).
5. `Rating.source`는 "onboarding"에서 갱신되거나 유지될 수 있으나, 점수는 새 값으로 반영된다.
6. 평균 평점 즉시 갱신.
7. CF 즉시 재계산.

> **AC 연결**: F-002 — "온보딩에서 선택한 로스터리를 이후 직접 재평가하면, 기존 온보딩 5점이 새 점수로 덮어쓰인다. 재평가 UI에는 기존 점수(5점)가 선택된 상태로 표시된다."

**QA 검증 항목**:
- [ ] 재평가 UI에서 기존 점수 5점이 미리 선택된 상태로 표시된다.
- [ ] 저장 후 Rating.score = 3으로 갱신됨을 DB에서 확인.
- [ ] Rating 레코드 수가 증가하지 않는다 (기존 레코드 업데이트, 신규 생성 없음).

### 4-I. 비로그인 사용자의 평가·즐겨찾기 시도

3. 로그인 페이지로 리디렉션된다.
4. 로그인 완료 후 원래 의도한 동작(평가/즐겨찾기)을 이어서 수행할 수 있다.

> **AC 연결**: F-005 — "비로그인 사용자가 평가 버튼을 클릭하면 로그인 페이지로 리디렉션된다." / F-007 — "비로그인 사용자가 즐겨찾기 버튼을 클릭하면 로그인 페이지로 리디렉션된다."

### 4-J. 온보딩 미완료 사용자의 평가·즐겨찾기 시도

**시나리오**: 로그인은 되었으나 온보딩 미완료(`onboardingVersion = null`) 상태에서 평가 또는 즐겨찾기 시도.

1. 로그인 상태에서 로스터리 상세 페이지로 URL 직접 접근한다.

2. 평가 버튼 또는 즐겨찾기 버튼을 클릭한다.

4. 설문 완료 후 홈 피드로 이동한다.

> **AC 연결**: F-001 — "온보딩 미완료 사용자가 로그인 상태에서 평가, 즐겨찾기 등 로그인 필수 기능에 접근하면 온보딩 설문 페이지로 강제 이동된다." / "로스터리 목록 및 상세 페이지는 온보딩 미완료 사용자 모두 접근 가능하며, 온보딩 강제 리디렉션 대상에서 제외된다."

- [ ] 온보딩 미완료 + 로그인 상태에서 평가 API 직접 호출 시도 → 차단 또는 온보딩으로 리디렉션.
- [ ] 온보딩 완료 후 이전 의도한 기능(평가/즐겨찾기)으로 자연스럽게 복귀하는지 확인.

### 4-L. 필터 결과 0개 상태

**시나리오**: 복합 필터 적용 결과 해당 조건의 로스터리가 없는 경우.

2. 해당 조건의 로스터리가 0개.
3. 빈 화면이 아닌 안내 문구가 표시된다 (예: "해당 조건의 로스터리가 없습니다. 필터를 변경해보세요.").
4. "필터 초기화" 버튼 클릭 시 모든 필터가 해제되고 전체 목록이 표시된다.

> **AC 연결**: F-004 — "적용된 필터 조건에 맞는 로스터리가 0개일 때 빈 화면이 아닌 안내 문구가 표시된다." / "필터 초기화 버튼 클릭 시 모든 필터 조건이 해제되고 전체 목록이 표시된다."

**QA 검증 항목**:
- [ ] 결과 0개 시 빈 흰 화면이 아닌 안내 문구가 표시된다.
- [ ] 필터 초기화 후 전체 목록 표시.
- [ ] 디카페인 원두가 없는 로스터리에서 "디카페인 가능" 필터만 적용 시 0개 안내 문구 표시.

### 4-M. 디카페인 토글 + 추천 결과 0개

**시나리오**: CF 활성화 상태에서 "디카페인만 보기" 토글을 켰을 때 해당 조건의 추천이 없는 경우.

1. 홈 피드에서 "디카페인만 보기" 토글을 활성화한다.
2. "새로운 로스터리" 섹션: `hasDecaf=true` 추천 0개 → 섹션 미표시.
3. "또 사고 싶은 로스터리" 섹션: `hasDecaf=true` 추천 0개 → 섹션 미표시.
4. 두 섹션 대신 "현재 디카페인 원두를 보유한 추천 로스터리가 없습니다." 안내 문구가 표시된다.

> **AC 연결**: F-006 — "'디카페인만 보기' 토글 활성화 시 두 섹션 모두 해당하는 로스터리가 0개이면, 섹션 대신 '현재 디카페인 원두를 보유한 추천 로스터리가 없습니다.' 안내 문구가 표시된다."

**QA 검증 항목**:
- [ ] 디카페인 토글 ON + 두 섹션 0개 시 명시된 안내 문구가 표시된다.
- [ ] "새로운 로스터리"만 0개이고 "또 사고 싶은"에는 결과가 있을 경우, 0개 섹션만 미표시.
- [ ] 토글을 다시 끄면 원래 CF 섹션이 복원된다.

### 4-N. 배치 계산 실패 폴백

**시나리오**: 새벽 3시 배치 작업 실패로 Recommendation 테이블에 데이터가 없는 경우.

1. 배치 작업이 실패하여 Recommendation 테이블이 비어 있다.
2. CF 활성화 조건(평가 3개 이상)을 만족하는 사용자가 홈 피드에 접근한다.
3. Recommendation 테이블에서 해당 사용자의 추천 데이터를 찾을 수 없다.
4. "새로운 로스터리", "또 사고 싶은 로스터리" 섹션 표시 불가.
5. "실시간 인기 로스터리" 섹션이 대체 표시된다 (전체 평균 평점 상위 5개, 동점 시 총 평가 수 많은 순).

> **AC 연결**: F-006 — "배치 계산 실패로 Recommendation 테이블에 데이터가 없는 경우, '실시간 인기 로스터리' 섹션으로 대체 표시한다." / "CF를 사용할 수 없는 모든 경우에 '실시간 인기 로스터리' 섹션이 표시된다."

**QA 검증 항목**:
- [ ] 배치 실패 시 홈 피드에 에러 화면이 표시되지 않는다.
- [ ] "실시간 인기 로스터리" 5개가 정상 표시된다.
- [ ] 배치 재실행 후 다음 홈 피드 로드에서 CF 섹션으로 복귀된다.

## 5. 이벤트 로깅 연결 표

아래 표는 각 시나리오에서 발생해야 하는 이벤트 로깅을 정리한다. QA 단계에서 이벤트 발생 여부와 파라미터 정확성을 검증한다.

| 시나리오 | 이벤트명 | 필수 파라미터 | 연결 KPI |
|---------|---------|-------------|---------|
| 2-A 단계 5 (추천 클릭) | `recommendation_clicked` | userId, roasteryId, recommendationSource="cf", timestamp | North Star, 추천 CTR |
| 2-A 단계 6 (구매 링크 클릭) | `purchase_link_clicked` | userId, roasteryId, timestamp | 추천→구매 전환율 |
| 2-B 단계 3 (평가 작성) | `rating_submitted` | userId, roasteryId, score=4, isFirstRating=true, timestamp | 첫 평가 완료율 |
| 2-C 단계 2 (온보딩 완료, Q4=FIRST_TIME) | `onboarding_completed` | userId, q5RoasteryCount=0, q4IsFirstTimeBuyer=true, onboardingVersion=3, timestamp | 온보딩 완료율 |
| 2-A 단계 3 (온보딩 완료, Q5 3개) | `onboarding_completed` | userId, q5RoasteryCount=3, q4IsFirstTimeBuyer=false, onboardingVersion=3, timestamp | 온보딩 완료율 |
| 4-G (평가 삭제) | 별도 이벤트 없음 (rating_submitted로 수렴하지 않음) | — | 내부 분석용 로깅 검토 권장 |
| 폴백 추천 클릭 (4-N 등) | `recommendation_clicked` | userId, roasteryId, recommendationSource="fallback_global", timestamp | North Star (분모 포함) |

> **참고**: `recommendationSource` 값은 `cf` / `fallback_global` 중 하나. MVP에서 폴백은 "실시간 인기 로스터리" 단일로 통합되어 모두 `fallback_global`을 사용한다 (v1.28).
